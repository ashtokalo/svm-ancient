 1. Что такое Files Coder.
────────────────────────────────────────────────────────────────────────────
   Files  Coder  -  это  небольшая текстовая утилита. С её помощью Вы можете
 производить  перекодировку  текстовых файлов из одной системы в другую. При
 этом  вы  можете  работать  с  такимим кодировками, которые отводят под код
 символа 1 или 2 байта.


 2. Как использовать Files Coder.
────────────────────────────────────────────────────────────────────────────
   Files  Coder  предназначен  для  работы  в  коммандной строке. В качестве
 параметров  Вы  должны  указать  имя файла содержащего информацию о системе
 кодировки,  имя  исходного  файла,  имя  которое  надо присвоить созданному
 файлу. При этом Вы должны соблюдать указанный выше порядок параметров.
   При  задании  параметров,  последний  параметр  можно  опустить, при этом
 созданный  файл  будет  иметь  имя, составленное из имени исходного файла +
 расширении заданное системой кодировки.
   В процессе работы, программа выдаст сообщение о количестве перекодируемых
 символов, а также будет показывать количество выполненной работы выраженной
 в   процентах.  По  окончанию  работы  программа  сообщит,  как  называется
 созданный файл.


 3. Что такое Files Coder Table.
────────────────────────────────────────────────────────────────────────────
   Программа   Files  Coder  не  предназначена  для  перекодировки  какой-то
 конкретной  системы  символов.  По  этой  причине,  информация  о  том  как
 производить  перекодировку храниться вовнешних файлах или Files Coder Table
 (таблица для Files Coder).
   Эти  файлы  хранят информацию, о кодах символов, которые надо заменять, и
 соответственно о кодах, на которые надо заменять эти символы.
   Кроме  того,  эти  файлы  содержат  информацию  о  типе системы символов,
 расширение  которе  добавляется  к имени результирующего файла, если тот не
 указан. Именно имя такого файла (Files Coder Table) необходимо передавать в
 качестве первого параметра в коммандной строке.


 4. Как создать свою Files Coder Table.
────────────────────────────────────────────────────────────────────────────
   При создании Files Coder Table надо учитывать следующие факты:
   1.  Необходимо  задать  тип  исходной  кодировки.  Для этого надо сделать
   следующее,  в файл, в котором будет находиться Files Coder Table добавить
   строку:

       Source char size=n

   причем,  вместо  n  подставить  1 или 2, что соответствует - один или два
   байта на один символ исходной кодировки.

   2. Необходимо задать тип результирующей кодировки. Для этого надо сделать
   следующее,  в файл, в котором будет находиться Files Coder Table добавить
   строку:

       Destenation char size=n

   причем,  вместо  n  подставить  1 или 2, что соответствует - один или два
   байта на один символ результирующей кодировки.

   3. Необходимо задать расширение, которое будет использовано при генерации
   имени для результата. Для этого надо сделать следующее, в файл, в котором
   будет находиться Files Coder Table добавить строку:

       Default extenation=ext

   причем, вместо ext подставить имя расширения без точки.

   4.  После этого можно задать несколько строк, которые будут добавляться в
   начало  создаваемого  файла.  Для этого надо сделать следующее, в файл, в
   котором будет находиться Files Coder Table добавить строку:

       Begin string=<text of string>

   причем, вместо <text of string> необходимо вставить нужнйю строку, длинна
   которой не должна привышать 80 символов.

   5.  После этого можно задать несколько строк, которые будут добавляться в
   конец  создаваемого  файла.  Для  этого надо сделать следующее, в файл, в
   котором будет находиться Files Coder Table добавить строку:

       End string=<text of string>

   причем, вместо <text of string> необходимо вставить нужнйю строку, длинна
   которой не должна привышать 80 символов.

   6. После того, как Вы выполнили предыдущие пункты (1-3 - обязательно, 4-5
   -  по  желанию) необходимо добаивить самую главную информацию - кодироки.
   Для  этого  надо  сделать  следующее,  в файл, в котором будет находиться
   Files   Coder   Table  добавить  строки,  сформатированные  по  следующей
   структуре:

       <source char code>=<destenation char code> ; <comment>

   причем,  вместо  <source char code> необходимо подставить код символа (из
   исходной  кодировки), который будет замещаться на <destenation char code>
   -  код  символа  в  результирующей  кодировке. При подставлении вы можете
   использовать   числа  диапозона  0..32760,  для  задания  кода  символа в
   конкретной  кодировке.  Не  указанные  символы  - не будут изменяться при
   перекодировке.  Между  кодами символов не должно находиться ничего, кроме
   символа  равно (3Dh в DOS кодировке). После кода символа в результирующей
   кодировке  Вы  можете поставить знаки - пробел и точка с запятой - (20h и
   3Bh  в DOS кодировке соответственно) после которых Вы можете писать любые
   коментарии.

   7.  Если  Вы  хотите вставить в файл свои коментарии Вы должны в качестве
   первых  символов  строки использовать знак - точка с запятой - (3Bh в DOS
   кодировке) или два знака - деления - (2Fh в DOS кодировке), после которых
   может  располагаться  любой текст. Если Вы хотите вставить текст без этих
   символов,  Вы  должны  явным  образом  указать  конец файла, для этого Вы
   должны  в  файл,  в  котором  будет находиться Files Coder Table добавить
   строку:

       end

   Встретив  такую  строку,  программа  закончит разбор файла, поэтому после
   этой команды может находиться любой текст.

   8.  При  создании  Files  Coder  Table, Вы должны учитывать тот факт, что
   любая  команда,  код  символа  или символы коментария должны начинаться с
   первого   символа  строки,  т.е.  перед  ними  не  должны  присутствовать
   какие-либо символы.


 5. Комплектация.
────────────────────────────────────────────────────────────────────────────
   В архиве с Files Coder находяться следующие файлы:
   fcoder.exe   - собственно Files Coder
   fcoder.doc   - этот файл Вы сейчас читаете
   alt2win.fct  - Files Coder Table для перекодировки DOS     -> Windows
   alt2gost.fct - Files Coder Table для перекодировки DOS     -> ГОСТ
   alt2koi.fct  - Files Coder Table для перекодировки DOS     -> KOI-8
   win2alt.fct  - Files Coder Table для перекодировки Windows -> DOS
   gost2alt.fct - Files Coder Table для перекодировки ГОСТ    -> DOS
   koi2alt.fct  - Files Coder Table для перекодировки KOI-8   -> DOS

   Если Вы создадите свою Files Coder Table не полинитесь прислать её мне, а
 я выложу её на свой сайт, откуда любой сможет её загрузить.


 6. Тестирование.
────────────────────────────────────────────────────────────────────────────
   Программа  тестировалась на  286-16 с  1  мегабайтом  памяти,  результаты
 удовлетворят  кого-угодно  -  для  перекодировки  файла  (тексты приколов с
 украинских BBS-ок) размером 429601 байт понадобилось 11 секунд.
